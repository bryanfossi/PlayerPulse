# UX Audit Report: PlayerPulse — May 2026

## Executive Summary

PlayerPulse is a well-architected recruiting CRM with strong branding and a clear value proposition, but five key friction points threaten soft-launch success:

1. **Subscription wall blocks free-tier access to full onboarding** (P0) — Athletes can start onboarding but hit a paywall mid-flow with no clear "free plan" alternative; messaging says "free forever plan available" on landing but doesn't reach signup.
2. **Empty states lack navigation guidance** (P1) — Schools board, communications log, and offers page show minimal helpful CTAs when empty, forcing users to hunt for next steps.
3. **Form validation UX inconsistent** (P1) — Some fields fail silently or show errors inline without helpful context; password validation message is unclear for 16-year-olds ("Enter 3–8" for height feet, but no visual picker).
4. **Public profile credibility gaps** (P2) — Sections with no data (awards, stats, schedule) still render headers, diluting visual hierarchy; missing headshot placeholder is prominent.
5. **Button/loading states lack affordance clarity** (P2) — `disabled:opacity-60` makes buttons appear faded but still interactive-looking; no skeleton loaders or optimistic UI for async operations.

**Launch recommendation:** Fix P0+P1 items before test phase (paywall logic, empty state CTAs, form UX). P2 items in follow-up sprint.

## Quick Wins (Ship This Week)

1. **Fix paywall messaging** (1h): Update `/onboarding/subscribe/page.tsx` to clarify free tier (25 schools, 3 credits/mo). Add "Free" plan card below Pro. Link from landing page hero button directly to free tier onboarding.

2. **Standardize empty states** (3h): Create reusable `EmptyStateCard` component with icon, heading, description, primary CTA. Use across schools, communications, offers pages. Test on mobile.

3. **Sync password validation** (0.5h): Change login min length from 6 to 8. Add test cases.

4. **Add confirmation dialog for destructive actions** (2h): Toast or modal before marking school "Declined" or removing parent invite. Test recovery flow if user dismisses.

5. **Disable button affordance fix** (1h): Update `disabled:opacity-60` → `disabled:opacity-50 disabled:cursor-not-allowed` in base button component. Visual test.

**Total: ~7.5 hours**

## Strategic Bets (Multi-Week)

1. **Mobile-first nav UX** (5h, high leverage): Add sticky breadcrumb trails to detail pages (school, profile edit, comms log). Redesign MobileNav to match desktop sidebar hierarchy. Test full user journey on 3 devices (iPhone 13, Samsung Galaxy A12, iPad).

2. **Public profile visual polish** (8h, soft-launch impact): Conditionally render sections only with data. Fix inline CSS inconsistencies. Add professional headshot placeholder (athlete-grade template). Test credibility with coaches (external feedback).

3. **Form UX for teens** (4h, retention): Replace text height input with visual stepper or numeric picker. Rewrite all form validation errors in plain language. Add field hints (e.g., "GPA as of last report card"). Field-by-field user testing with 2–3 athletes.

4. **Skeleton loaders & optimistic UI** (5h, perceived performance): Add skeleton cards for dashboard stats, schools, comms. Use optimistic updates for school status changes, contact logs. Implement React Suspense streaming per section.

5. **AI trust & edit loop** (3h, feature stickiness): Add word count and confidence score to drafted emails. Improve regenerate UX with before/after preview. Log which emails coaches reply to (momentum signal).
